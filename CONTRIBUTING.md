# Contributing

## Before starting

Every change must be linked to a GitHub Issue with a clear outcome and acceptance criteria. Product copy must identify whether a statement is verified, provisional, or intentionally omitted.

## Branches

- `feat/<issue>-short-description`
- `fix/<issue>-short-description`
- `content/<issue>-short-description`
- `chore/<issue>-short-description`

## Commits

Use concise conventional commits:

- `feat: add product filter model`
- `fix: restore mobile navigation focus`
- `content: add TG series overview`
- `docs: document image approval flow`

## Pull requests

- Open a draft PR early.
- Keep one concern per PR.
- Include screenshots for visual work.
- Include the related Issue using `Closes #123` when appropriate.
- Confirm desktop, mobile, keyboard, content, and link checks.
- Do not merge with failing required checks.

## Definition of done

- Acceptance criteria are met.
- Automated checks pass.
- Responsive behaviour is reviewed.
- Keyboard and visible-focus behaviour are reviewed.
- Public product claims have a source or are removed.
- Images have useful alternative text and usage rights.
- Documentation is updated when behaviour or process changes.
