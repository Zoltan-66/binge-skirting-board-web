# Development Workflow

1. Select an Issue in `ready` state.
2. Create a focused branch from `main`.
3. Open a draft PR early.
4. Pass CI and complete responsive, accessibility, and content reviews.
5. Squash merge after approval.

Status labels:

- `status: triage`
- `status: ready`
- `status: in progress`
- `status: review`
- `status: blocked`

Every visual PR includes screenshots. Every product-content PR identifies source evidence and claim state.
