# Security Policy

## Reporting

Do not open public Issues for vulnerabilities, leaked credentials, private customer information, or sensitive RFQ data. Contact the repository owner privately through GitHub.

## Supported version

Only the current production branch is supported.

## Baseline

- Secrets must never be committed.
- RFQ uploads must be validated, size-limited, and stored privately.
- Dependencies are reviewed through automated checks.
- Production forms require rate limiting, spam protection, and privacy notice acceptance.
